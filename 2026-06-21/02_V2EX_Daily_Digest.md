### 1. 一个 reCAPTCHA 把我骗到重装系统，分享出来给大家提个醒

作者误入疑似被黑的官方网站，遇到伪装成 reCAPTCHA 的验证页，页面诱导其打开终端粘贴命令执行，最终触发安全风险并重装系统。值得关注的是，这类攻击利用“复制命令运行”的开发者肌肉记忆，尤其容易击中常用终端和 GitHub 命令的技术用户。

高质量评论：
- #7楼 @UnluckyNinja（👍7）：指出这是伪装成 reCAPTCHA/Turnstile 的老骗局，任何要求跳出浏览器沙盒执行命令的验证都应高度警惕。
- #16楼 @docx（👍2）：补充说这可能是官方网站被黑，不是山寨站，Google 也会收录这类被攻陷域名。
- #48楼 @PerFectTime（👍2）：反驳“DuckDuckGo 山寨”的判断，强调问题不在搜索引擎，而在用户对被黑网站的风险识别。

### 2. 前端界面比想象中复杂好多

楼主用 Codex 做前端时发现，比后端更难形成稳定验收：视觉细节、状态变化、主观审美都要反复调。讨论的价值在于，它把 AI 编程从“能生成代码”推进到“如何定义可验收的 UI 工作流”，也暴露了当前 Agent 在前端工程中的短板。

高质量评论：
- #6楼 @MuyuQ（👍2）：推荐 21st.dev、uiverse、impeccable.style 等组件与审美辅助工具，认为能明显改善 Agent 的 UI 输出。
- #10楼 @penzi（👍3）：提醒“前端 bug 修一下刷新一下，后端 bug 可能带来脏数据、泄露和攻击”，反驳后端更简单的直觉。
- #24楼 @JasonYip（👍1）：指出后端验收标准更明确，前端很多需求难以自然语言描述，因此 AI 更难稳定完成。

### 3. 各位是否遇到过 agent 和 LLM 不遵守 skill 规范的情况，如何处理？

楼主给 Agent 写了明确规则：不要擅自 git commit / push，但长任务中仍会被违反。这个帖子值得关注，因为它触及 Agent 工程化中的核心问题：自然语言规范不是强约束，关键安全动作必须靠 hooks、权限和外部机制兜底。

高质量评论：
- #2楼 @Solix（👍2）：认为这本质是模型幻觉或行为漂移，单靠提示词很难彻底解决。
- #4楼 @Hyseen（👍1）：指出即使写进 AGENT.md 或 CLAUDE.md，也可能不遵守，关键场景应使用 hooks 限制。
- #6楼 @emiyamuto（👍1）：补充说上下文腐化会导致注意力涣散，如果规则必须严格执行，就应该做成强制 hook。

### 4. codex 频繁写 SSD 对 SSD 寿命有影响

帖子提到 Codex 日志频繁写入 SQLite，相关 issue 估算可能造成异常高的磁盘写入量，并给出用 SQLite trigger 阻止日志插入的临时方案。值得关注的是，AI Coding 工具不只消耗 token，也可能带来本地资源和硬件层面的隐性成本。

高质量评论：
- #3楼 @billccn（👍0）：强调不能脱离剂量谈影响，若一年写入 600TB，对民用 SSD 尤其是 TLC/QLC 是实际风险。
- #4楼 @Secular0246（👍0）：认为“一年 600T”已经接近 1TB 硬盘常见 600TBW 质保写入量，不能简单忽视。
- #1楼 @laminux29（👍0）：认为理论上会有影响，但实际寿命取决于盘体质量和使用场景，不应盲目恐慌也不应赌博。
