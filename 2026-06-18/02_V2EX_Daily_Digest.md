### 1. Claude Code 实践心得：多 Agent 最大的问题是“漂移”
原帖讨论 Claude Code 与 multi-agent 编程框架的真实体验：作者认为单个 AI 已能承担大量编码，但多 Agent 链式协作会带来需求传递失真，最后生成“看起来对、内部一坨”的代码。值得关注的是，这不是简单吐槽工具，而是在讨论 AI 编程产品的核心设计：人类如何保持透明、可控、可审核。

💬 热评：
- #7楼 @GeruzoniAnsasu (👍1) 认为 LLM 的“输出即思维”决定了外部流程治理很难弥补模型底层能力，Agent 产品真正价值应是设计更适合人类掌控的协作模式。
- #1楼 @lolo1 (👍0) 赞同不要滥用角色型流程 Agent，信息传递会持续失真，单个 Claude Code 能解决就不要拆成多个 Agent。
- #3楼 @ximaoyang (👍0) 补充真实工程不是从零做商城，而是在长期系统中处理小 ticket，多 Agent 往往杀鸡用牛刀，反而增加审核负担。

GLM 5.2 已开源权重并上架 OpenRouter，帖子聚焦社区第一手体验。评论整体偏正面：不少人认为它作为开源模型“很能打”，甚至有人拿它对标 Opus 级别体验，但也指出速度慢、请求拥堵等问题。值得关注的是，国产开源大模型正在进入真实 Agent/编码工具对比，而不只是参数和榜单讨论。

- #5楼 @cyp0633 (👍0) 评价“作为开源模型非常令人惊喜”。
- #3楼 @NathanInMac (👍0) 指出主要短板是太慢，除此之外体验不错。

### 4. Intel NPU 到底有什么实际作用？
一位用户发现新电脑任务管理器里多了 Intel NPU，但几乎看不到负载，因此询问实际价值。讨论集中在当前 PC 端 NPU 的尴尬处境：它对 LLM 帮助有限，更适合摄像头背景虚化、OCR、Windows Recall 等轻量 AI 场景。值得关注的是，AI PC 硬件卖点和真实应用之间仍存在明显断层。

💬 热评：

- #3楼 @peteretep (👍0) 认为目前主要是跑分好看，本地模型还没有很好的消费级场景。
- #5楼 @nsjs (👍0) 指出 NPU 对 LLM 帮助不大，更偏图像、OCR 等任务，Recall 还伴随隐私争议。

### 5. Linux 个人服务器如何做安全监控？
帖主提到 VPS 曾被植入 rootkit，云厂商安全面板能及时告警，但自建 Home Server 缺少类似能力，因此求推荐 Linux 安全监控方案。评论没有给出单一神器，反而强调更基础的安全策略：减少暴露面、VPN 访问后台、容器隔离、观察异常资源占用。值得关注的是，个人服务器安全问题正在从“装软件”转向“架构隔离和运维习惯”。

💬 热评：
- #1楼 @NamelessRain (👍0) 建议只开放必要端口，用 OpenVPN 做内外网隔离，减少直接暴露。
- #3楼 @loading (👍0) 认为如果担心具体应用漏洞，应该优先用 Docker 隔离。
- #5楼 @Lentin (👍0) 提醒没有绝对安全，即使不开端口也可能遇到供应链投毒，平时要关注 CPU、网络等异常。
