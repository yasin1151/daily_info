1. 公司 vibe coding 的项目，团队已经无法掌控了  
摘要：一家公司用 Agent 快速堆出客服系统后，需求、代码质量、测试和排障逐步失控。帖子核心价值在于讨论 AI 写代码进入生产后，团队如何重新建立需求边界、测试体系和代码可见性。  
精选评论：  
- 85 楼（0 赞）：现实是 vibe coding 在生产里其实更需要有人能把需求、边界和验收标准讲清楚，否则 AI 只是把混乱更快地写进代码库。  
- 57 楼（0 赞）：任何可复现异常都应该沉淀成测试用例，再让 AI 针对性修复；没有测试量支撑，性能和稳定性问题迟早会失控。  
- 23 楼（0 赞）：AI 生成代码常常语法、lint、单测都过，真正问题在业务语义、跨模块副作用和团队缺乏对生成代码的可见性。  
原帖：https://www.v2ex.com/t/1224558

2. 学 Linux，不一定要从安装系统开始（写给 Linux 初学者的最佳实践）  
摘要：帖子反对把“装系统”作为 Linux 入门第一步，主张先从可用终端环境、文件系统、权限、进程和命令行习惯开始。适合关注开发者教育、工程训练路径的人参考。  
精选评论：  
- 3 楼（0 赞）：十多年 Linux 桌面用户建议小白可先从 Windows PowerShell 或 macOS 终端开始，先习惯命令行，再逐步接触 Busybox、Bash、Termux 和手动装 Arch。  
- 1 楼（0 赞）：反方认为系统安装本身已高度图形化，不该被视作过高门槛；Ubuntu live 环境和现代发行版安装流程已经足够简单。
