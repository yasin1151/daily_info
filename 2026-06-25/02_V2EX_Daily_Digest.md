3. **求使用 AI 从 0 到 1 开发一个完整 web 前后端项目的成功经验**

摘要：楼主尝试先让 Claude 生成 PRD、计划和前端方案，再让 AI 实现完整项目，但结果只搭出架子，修改时反而要读懂 AI 的实现方式。这个帖子值得关注，因为评论集中给出了“AI 不是替你架构项目，而是复制你已建立的工程范式”的实践经验。

精选评论：
- 0 赞 / 1 楼：前端、后端、基础 CRUD、技术架构选项、项目搭建，全部自己手动完成。然后把核心思路、技术栈使用生成项目专用 skill。一定要先手写一个模块功能，然后让 AI 去抄，否则它自由发挥太多。
- 0 赞 / 3 楼：先启动一个 agent 确定需求，生成 README 和 TODO；创建前后端子目录；再让一个 agent 生产和验任务，另一个从 TODO 里取任务消费；最后 Codex 重构 UI，DeepSeek 写中文文档。
- 0 赞 / 6 楼：让 agent 根据需求和技术栈写 todo.md，拆分成多个 phase，之后一个个让其它 agent 完成，最后多用几个 agent 检查 BUG 和未完成的功能。文档越少越好，重点是告诉它做什么，不要教它怎么做。
