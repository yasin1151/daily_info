1. **几年前，大家会想到今天的 AI 那么离谱吗？**

摘要：楼主回顾从 2023 年开始使用 ChatGPT、Claude 等工具的体验，认为 AI 的变化速度远超预期。讨论重点不是单纯感叹模型变强，而是 AI 已经进入数据处理、SQL、视频创作、代码辅助等真实工作流，同时也暴露出企业业务逻辑、遗留系统和实际落地之间的鸿沟。

高质量评论：
- 12 楼（6 赞）：老板天天念叨 AI 多牛逼，但公司 ERP 十几年历史，甲方乙方和大量业务逻辑不是一句 AI 就能理解，真实企业系统仍然很难被快速替代。
- 1 楼（0 赞）：分享从 2023 年中开始使用 ChatGPT 的经历，最初主要用于清洗数据和处理 SQL，短 SQL 效果不错，复杂任务仍有限制。
- 3 楼（0 赞）：提到早期围绕 GPT 做内容和“调教”的经历，说明生成式 AI 在中文社区的传播从工具尝鲜很快进入内容生产。

原帖：https://www.v2ex.com/t/1225706

2. **Fable 5 延期到 7 月 12 号**

摘要：Claude Fable 5 促销访问延期，但楼主指出它消耗额度很快，做一次项目级 code review 可能就要花掉上百美元。值得关注的是，AI 编程模型的“可用性”已经不只看效果，还要看额度、计费、上下文成本和团队预算能否支撑日常开发。

高质量评论：
- 2 楼（5 赞）：延期并不等于重置额度，很多人可能已经把本周额度用完，只能等重置后再继续使用。
- 1 楼（0 赞）：如果只是延长活动时间但不重置额度，用户会在额度恢复后集中使用，之后若继续延期仍会出现类似尴尬。

原帖：https://www.v2ex.com/t/1225713

3. **使用 Claude Code 进行开发，一天用 600 人民币算多吗？**

摘要：帖子讨论 Claude Code 高强度使用下的真实成本，有人一天花费数百元甚至更多。它反映出 AI 编程工具从“订阅软件”变成“按算力和 token 消耗计费的生产资料”，开发者和团队需要重新评估代码生成、上下文复用、测试生成与人工审查之间的投入产出比。

高质量评论：
- 96 楼（2 赞）：公司付费场景下过去 30 天用了 1300 美元，忙的时候一天 200–300 美元也可能发生，楼主的支出在重度使用中不算夸张。
- 85 楼（0 赞）：很多人对 API 价格缺乏直觉，自己一天 Codex 用到约 9300 万 token，换算到 Claude API 会是非常高的成本。
- 33 楼（0 赞）：回忆早期高额度、低风控时期，一天可生成大量代码和测试；但这也说明 AI 编程效率提升背后高度依赖额度、策略和模型状态。

原帖：https://www.v2ex.com/t/1225437

4. **火山引擎的后台做的也太乱了吧，找功能就像逛迷宫**

摘要：楼主首次使用火山引擎 TTS API，被免费额度、API Key、欠费通知和后台入口设计困扰。这个帖子值得产品工程读者关注：面向开发者的云服务不只是 API 能跑，还需要清晰的计费模型、权限/密钥管理和可理解的信息架构，否则会快速消耗信任。

高质量评论：
- 40 楼（3 赞）：认为火山引擎没有充分使用抖音、字节等品牌背书，整体给人的感觉不像是认真做开发者平台。
- 30 楼（0 赞）：指出火山后台学习成本极高，刚熟悉后又改新版 UI，功能逻辑重排，对开发者非常不友好。
- 4 楼（0 赞）：对比字节面向普通用户的产品几乎零学习成本，但面向开发者/创造者的火山、飞书、即梦等后台却常像迷宫。
